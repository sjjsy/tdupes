- Then the script should wait and ask whether the user has finished with editing the TSV
- Then it should again summarize the actions and ask whether to execute the keep/delete actions as per the TSV
- Then, if "yes" is given, then it should trash all the files with the 'gio trash' utility (such that they are still recoverable until the cans are emptied)
- Then it should summarize the outcome
- All summaries should include the current size consumed by the files and the total size that could be redeemed by following the proposed or already executed actions

- Make this into a proper utility with a usage documentation that is given with '-h', '--help', and unsupported flags; And make it such that it could be published in git but also in a public utility index that would allow other people using Ubuntu to easily install it
- When the script starts it should check whether all the CLI utilities or Python modules are available and fail gracefully and informatively if there are problems
- By default the tool is somewhat verbose but with a verbosity flag it is more verbose and with '-q' or '--queit' it is silent
